**Last Updated:** 30 June 2026

# General

## What is Vikasa Network?

Vikasa Network is a Polygon-based Web3 ecosystem designed to encourage genuine user participation through blockchain technology, community engagement, and digital rewards.

The ecosystem is powered by the **VIK** utility token and aims to simplify Web3 adoption through a mobile-first platform.

---

## What is VIK?

VIK is the native utility token of the Vikasa Network ecosystem.

It is designed to support ecosystem participation, community rewards, platform activities, and future utility features.

---

## Which blockchain does VIK use?

VIK is deployed on the **Polygon Proof-of-Stake (PoS)** network as an ERC-20 compatible utility token.

---

## Why was Polygon selected?

Polygon provides low transaction fees, fast confirmations, and compatibility with the Ethereum ecosystem, making it suitable for a scalable Web3 rewards platform.

---

## Can additional VIK tokens ever be created?

No.

The entire token supply was created during deployment. The deployed smart contract does not include a public mint function for creating additional VIK.

---

## Is the smart contract publicly verified?

Yes.

The source code is publicly verified on PolygonScan.

---

## Has the smart contract been independently audited?

Not yet.

The VIK smart contract has **not yet undergone an independent third-party security audit**.

Current public verification includes:

* Verified source code on PolygonScan
* GoPlus Security automated analysis
* QuickIntel Security Scan
* OpenZeppelin ERC-20 implementation
* Public GitHub documentation

An independent audit may be considered as the ecosystem continues to mature.

---

## What do GoPlus Security and QuickIntel mean?

GoPlus Security and QuickIntel provide automated smart contract analysis to identify common security risks.

These reports improve transparency but **do not replace an independent professional security audit**.

Users should always perform their own research before interacting with any blockchain project.

---

## Is the contract ownership renounced?

No.

Contract ownership has not been renounced.

Users should verify current ownership and contract permissions directly on PolygonScan before interacting with the token.
